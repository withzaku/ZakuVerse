# Technical SEO Operator Checklist

Use this checklist when running crawl and indexation remediation cycles.

## 1. Scope and Priorities

- Define business-critical URL groups before making technical changes.
- Confirm conversion routes and proof routes are included in priority set.
- Capture baseline crawl share and index quality for 14-day and 28-day windows.

## 2. Crawl Diagnostics

- Segment bot hits by route group (service, proof, utility, low-value).
- Identify parameter patterns and stale paths consuming crawl budget.
- Check repeat crawl frequency on critical pages.

## 3. Route Hygiene

- Validate canonical consistency on primary templates.
- Remove redirect chains and broken redirect paths.
- Apply index controls to low-value routes where needed.

## 4. Internal Link Reinforcement

- Increase links from authority pages to high-value service pages.
- Link proof-case assets to corresponding service pages.
- Verify crawlable path depth for all priority routes.

## 5. Metadata and Structured Data

- Ensure metadata is consistent with route intent.
- Validate JSON-LD on core service and case pages.
- Recheck sitemap route coverage after updates.

## 6. Validation and Reporting

- Re-measure crawl allocation after deployment.
- Compare useful indexed URLs against baseline.
- Track inquiry quality and technical referral changes.
- Archive cycle notes with date, changes, and outcomes.
